# Fix Gradle Build Error

You got this error because the Gradle wrapper in the `android` directory is corrupted or incomplete.

## Solution: Clean Build

### Option 1: Remove Android Directory (Recommended)

The `android` directory was auto-generated and is causing issues. Remove it and let EAS Build create a fresh one:

```bash
cd /Users/hiteshrajbhotra/Documents/Production/FarmTime/mobile

# Backup first (optional)
mv android android.backup

# Or delete it
rm -rf android

# Now rebuild
eas build --platform android --profile preview --clear-cache
```

### Option 2: Fix Gradle Wrapper

If you want to keep the android directory:

```bash
cd /Users/hiteshrajbhotra/Documents/Production/FarmTime/mobile/android

# Re-download Gradle wrapper
./gradlew wrapper --gradle-version 8.0.1

# Or if gradlew doesn't work, regenerate it
cd ..
npx expo prebuild --clean
```

### Option 3: Use Managed Workflow (Easiest)

Remove the android directory completely and use Expo's managed workflow:

```bash
cd /Users/hiteshrajbhotra/Documents/Production/FarmTime/mobile

# Remove android directory
rm -rf android
rm -rf ios

# Clear EAS cache
eas build --platform android --profile preview --clear-cache
```

## Why This Happens

The error occurs when:
1. You ran `expo prebuild` or `npx expo run:android` before
2. This created the `android` directory locally
3. The Gradle wrapper files got corrupted or incomplete
4. EAS Build tries to use these files and fails

## Best Practice

For Expo projects using EAS Build:
- ✅ **Don't commit** `android` or `ios` directories
- ✅ **Let EAS Build** generate them in the cloud
- ✅ **Use managed workflow** when possible
- ✅ **Add to .gitignore**: `android/` and `ios/`
